# Обучение и «саморазвитие» BuilderOS

## Что НЕ делает локальная LLM

Веса модели Ollama **не дообучаются** во время работы. Это нормально:
continuous fine-tuning тяжёлый, нестабильный и рискованный для ИП.

## Что делает система вокруг модели

| Механизм | Где | Эффект |
|---|---|---|
| Память фактов | Memory + «запомни» | Помнит предпочтения и факты |
| Diff шаблонов | версии шаблонов | PATTERN / TEMPLATE_HINT |
| RAG по нормам | Knowledge + Qdrant | Ответы по ГОСТ/СП |
| **Граф объекта** | Graph | Связи заказчик↔документы↔платежи |
| Hybrid LLM | Ollama → OmniRoute | Помощь в разборе фраз |

Умнеет **контекст**, а не нейросеть сама по себе.

## Можно ли спрашивать облачную модель «для учёбы»?

Да, как **учитель**, с правилами:

1. В облако — только обезличенный вопрос (без сканов, ИНН, сумм договора).
2. Ответ не становится истиной автоматически.
3. В Memory/Graph пишется проверенный **паттерн**.
4. Документы по-прежнему только из шаблонов.

Флаги: `LLM_CLOUD_FOR_KNOWLEDGE`, `LLM_CLOUD_FOR_VISION` — по умолчанию `false`.
Intent-классификация в hybrid может ходить в OmniRoute — это не обучение на ваших договорах.

## Следующий шаг (после графа)

Teacher-контур: локально «не уверен» → sanitized вопрос учителю → сохранение PATTERN
с `source=teacher` и привязкой к объекту/типу документа.
